# 🔧 دليل الإعداد وحل المشاكل - Pharma Qods

## ❌ المشكلة: "تعذر تحميل قاعدة البيانات الرئيسية"

هذه الرسالة تظهر عندما لا يتمكن النظام من العثور على ملف قاعدة البيانات.

---

## ✅ الحل السريع

### 1️⃣ **إنشاء المجلد والملف**

```
yahia preveé/quds site vol 5/
├── index.html
├── all-products.html
└── list/                    ← أنشئ هذا المجلد
    └── list.xlsx           ← ضع ملف الجرد هنا
```

### 2️⃣ **تشغيل عبر Web Server**

**خيار 1: استخدام Python (مثبت على Windows)**
```bash
cd "c:\Users\Gigahertz\Desktop\yahia preveé\quds site vol 5"
python -m http.server 8000
```
ثم افتح: `http://localhost:8000`

**خيار 2: استخدام Node.js**
```bash
npm install -g serve
serve -s "c:\Users\Gigahertz\Desktop\yahia preveé\quds site vol 5"
```

**خيار 3: استخدام VS Code Live Server**
- اضغط بزر اليمين على `index.html`
- اختر "Open with Live Server"

## 📁 هيكل المشروع الصحيح

```
quds site vol 5/
├── index.html              (الصفحة الرئيسية)
├── all-products.html       (عرض كل المنتجات)
├── list/                   (مجلد البيانات)
│   ├── list.xlsx          (ملف الجرد الرئيسي) ⭐ مهم
│   ├── A-01.xlsx          (ملف الفئة 1)
│   ├── B-02.xlsx          (ملف الفئة 2)
│   └── ...
```

---

## 🆕 الميزة الجديدة: رفع الملفات مباشرة

في الصفحة الرئيسية، ستجد زر **"📤 رفع list"** الذي يسمح لك برفع ملف الجرد مباشرة دون الحاجة لمجلد `list/`

---

## 📋 متطلبات ملف الجرد (list.xlsx)

### الصيغة الأساسية:
| اسم المنتج | الكمية |
|----------|--------|
| منتج 1    | 100    |
| منتج 2    | 50     |

### البدائل المدعومة:
- الأعمدة: `اسم المنتج` أو `Product Name` أو `Name`
- الكميات: `الكمية` أو `Quantity` أو `QTY`

---

## 🐛 اختبار وتشخيص

افتح **Developer Console** (F12) وستجد:
- ✅ **الملفات الموجودة** (علامة خضراء)
- ❌ **الملفات الناقصة** (علامة حمراء)

### مثال من Console:
```
✅ list/list.xlsx
✅ list/A-01.xlsx
❌ list/B-02.xlsx - لا يمكن الوصول إليه
```

## ⚠️ الأخطاء الشائعة

```
# ❌ خطأ - لن يعمل
file:///C:/Users/...../quds site vol 5/index.html
```

### ✅ الحل: استخدام localhost
```
# ✅ صحيح - يعمل بشكل صحيح
http://localhost:8000
```

---

### ❌ خطأ 2: اسم المجلد أو الملف مختلف

```
# ❌ خطأ
list/
├── inventory.xlsx     (اسم مختلف!)

# ✅ صحيح
list/
├── list.xlsx          (نفس الاسم المطلوب)
```

---

### ❌ خطأ 3: ملف Excel فارغ

تأكد من أن الملف يحتوي على:
- أسماء المنتجات في العمود الأول
- الكميات في أحد الأعمدة
- مسافة واحدة على الأقل من البيانات

---

## 💡 نصائح إضافية

### 🔄 إعادة تحميل قاعدة البيانات
اضغط على زر "Reload Main List" لإعادة تحميل البيانات

### 📤 رفع ملف جديد
اضغط على "📤 رفع list" لرفع ملف جرد جديد مباشرة

### 🔍 البحث عن فئة معينة
ضع ملف الفئة في مجلد `list/` باسم يطابق QR Code:
- QR: `A-01` ← الملف: `list/A-01.xlsx`
- QR: `B-02` ← الملف: `list/B-02.xlsx`

---

## 📞 الدعم

إذا استمرت المشكلة:
1. تحقق من **Developer Console** (F12) للأخطاء
2. تأكد من **صيغة الملف** (xlsx أو xls)
3. تأكد من **أسماء الأعمدة** في الملف
4. جرب **رفع ملف جديد** باستخدام الزر الجديد
